# General Python FAQ

## General Information

### What is Python?

Python is an interpreted, interactive, object-oriented programming language. It incorporates modules, exceptions, dynamic typing, very high level dynamic data types, and classes. It supports multiple programming paradigms beyond object-oriented programming, such as procedural and functional programming. Python combines remarkable power with very clear syntax.

### What is the Python Software Foundation?

The Python Software Foundation is an independent non-profit organization that holds the copyright on Python versions 2.1 and newer. The PSF's mission is to advance open source technology related to the Python programming language.

### Are there copyright restrictions on the use of Python?

You can do anything you want with the source, as long as you leave the copyrights in and display those copyrights in any documentation about Python that you produce. It's OK to use Python for commercial use, to sell copies of Python in source or binary form, or to sell products that incorporate Python.

### Why was Python created in the first place?

Guido van Rossum created Python during the 1989 Christmas holidays while working in the Amoeba distributed operating system group at CWI. He needed a scripting language with a syntax like ABC but with access to Amoeba system calls. He decided to create a generally extensible language.

### What is Python good for?

Python is a high-level general-purpose programming language. The language comes with a large standard library covering string processing, internet protocols, software engineering, and operating system interfaces. A wide variety of third-party extensions are also available at PyPI.

### How does the Python version numbering scheme work?

Python versions are numbered "A.B.C":
- A is the major version number
- B is the minor version number
- C is the micro version number (bugfix releases)

Additional suffixes: aN (alpha), bN (beta), rcN (release candidate).

### How do I obtain a copy of the Python source?

The latest Python source distribution is always available from https://www.python.org/downloads/. Development sources are at https://github.com/python/cpython/.

### How do I get documentation on Python?

Standard documentation is at https://docs.python.org/3/. The documentation is written in reStructuredText and processed by Sphinx.

### Is there a Python tutorial?

Yes. The standard documentation includes The Python Tutorial. Consult the Beginner's Guide at https://wiki.python.org/moin/BeginnersGuide.

### Why is it called Python?

When he began implementing Python, Guido van Rossum was also reading the published scripts from "Monty Python's Flying Circus". He thought he needed a name that was short, unique, and slightly mysterious.

## Python in the real world

### How stable is Python?

Very stable. New, stable releases have been coming out roughly every 6 to 18 months since 1991. As of version 3.9, Python has a new feature release every 12 months (PEP 602).

### How many people are using Python?

There are probably millions of users, though it's difficult to obtain an exact count.

### Have any significant projects been done in Python?

Yes. See https://www.python.org/about/success for a list. Companies that use Python internally include Google, Yahoo, and Lucasfilm Ltd.

### Is it reasonable to propose incompatible changes to Python?

In general, no. There are already millions of lines of Python code, so any change that renders even a fraction of existing programs invalid is frowned upon.

### Is Python a good language for beginning programmers?

Yes. Python's simple, readable syntax, interactive interpreter, and extensive standard library make it an ideal first language.
